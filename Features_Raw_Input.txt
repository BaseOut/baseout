For backups:

- In addition to frequency we will have either static or dynamic. Static means that they will get saved directly to a storage location as a CSV file whereas dynamic means they'll get saved into the database, which allows for more functionality. The lower tier might be static only and then higher tiers would support static or dynamic. If they do a dynamic backup then it will open up more functionality and more modules. If they do a static backup then it will limit what they can do but might be a more secure way because we don't store the data in a database.
- Also the database location and format is another one. Some tiers will support a SQLite database; others will be a hosted Postgres database and it will be not dedicated. - Another feature set of backups is the storage location for files or static data files. That would be hosted by us or allow you to bring your own storage location like Google Drive, Box, Dropbox, OneDrive, and so every tier will have different ones that we support. There will be limits on the amount of data storage that we would push.
