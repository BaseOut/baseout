# Iconography

The Baseout iconography system extends the geometric precision and layered philosophy of our logo into a comprehensive icon library. Our icons emphasize clarity, consistency, and functional communication.

## Design Principles

### Geometric Foundation
- **Based on 24px grid system**
- **2px stroke weight standard**
- **Rounded corners (2px radius)**
- **Consistent alignment and spacing**
- **Mathematical precision in construction**

### Visual Style
- **Outline style primary** (stroke-based)
- **Filled variants for emphasis**
- **Layered designs** reflecting brand metaphor
- **Minimal detail** for clarity at small sizes
- **Consistent optical sizing**

### Brand Alignment
- **Mountain/peak motifs** where appropriate
- **Layered elements** suggesting data stacks
- **Angular geometry** with soft edges
- **Professional and approachable** aesthetic

## Icon Specifications

### Standard Sizes
- **16px**: Small UI elements, dense interfaces
- **20px**: Standard buttons, form elements
- **24px**: Primary size, navigation, toolbars
- **32px**: Large buttons, featured elements
- **48px**: Headers, hero sections
- **64px**: Landing pages, major features

### Grid System
```
24px icon on 24px grid:
- 2px padding all sides
- 20px live area
- 2px stroke weight
- 2px corner radius
```

### Stroke Standards
- **Primary stroke**: 2px width
- **Secondary details**: 1.5px width
- **Fine details**: 1px width (sparingly used)
- **Stroke caps**: Rounded
- **Stroke joins**: Rounded

### Color Applications
- **Primary**: Baseout Teal (#2D5A5A)
- **Secondary**: Slate Gray (#5D6D7E)
- **Disabled**: Light Gray (#8E9AAF)
- **On dark**: Pure White (#FFFFFF)
- **Success**: Mountain Green (#3D6B3D)
- **Warning**: Peak Orange (#D17A3A)
- **Error**: Error Red (#E74C3C)

## Icon Construction Guidelines

### Geometric Construction
1. **Start with basic shapes** (circles, squares, triangles)
2. **Use grid alignment** for consistency
3. **Apply stroke weight** uniformly
4. **Add rounded corners** for softness
5. **Maintain optical balance** over mathematical precision

### Layered Design Approach
- **Foreground layer**: Primary element
- **Background layer**: Supporting context
- **Overlap areas**: Create depth and dimension
- **Consistent layer relationship** across icon family

### Detail Guidelines
- **Minimize fine details** at standard sizes
- **Focus on recognizable silhouettes**
- **Use negative space** effectively
- **Maintain clarity** at 16px minimum
- **Test legibility** across platforms

## Usage Guidelines

### Accessibility Standards
- **Minimum contrast ratio**: 3:1 against background
- **Alternative text**: Provide for screen readers
- **Size requirements**: 16px minimum for interactive elements
- **Touch targets**: 44px minimum for mobile
- **Color independence**: Icons should work without color

### Platform Considerations

#### Web/Digital
- Use SVG format for scalability
- Provide fallback PNG for older browsers
- Optimize file sizes for performance
- Ensure crisp rendering at all sizes
- Consider dark mode alternatives

#### Mobile Applications
- Provide multiple density versions (1x, 2x, 3x)
- Test on various screen sizes
- Consider thumb-friendly sizing
- Optimize for touch interactions
- Account for system icon conventions

#### Print Applications
- Use vector formats when possible
- Test legibility at print sizes
- Consider ink coverage for cost
- Provide high-resolution raster alternatives
- Account for paper and printing limitations

### Icon Combinations
- **Maintain consistent visual weight**
- **Use similar stroke widths**
- **Align to common baseline**
- **Preserve adequate spacing**
- **Group related functions visually**

## Custom Icon Creation

### Design Process
1. **Identify functional need**
2. **Research existing solutions**
3. **Sketch conceptual options**
4. **Create grid-based construction**
5. **Test at various sizes**
6. **Validate with users**
7. **Document usage guidelines**

### Quality Checklist
- [ ] Follows 24px grid system
- [ ] Uses consistent stroke weight
- [ ] Maintains visual balance
- [ ] Works at minimum size (16px)
- [ ] Aligns with brand aesthetic
- [ ] Clear and recognizable
- [ ] Accessible contrast ratios
- [ ] Optimized file size

### Brand Consistency
- **Reference existing icons** before creating new ones
- **Maintain stylistic coherence** across the system
- **Test alongside other icons** for visual harmony
- **Consider cultural implications** and universal understanding
- **Document design decisions** for future reference

## Icon Do's and Don'ts

### ✅ DO
- Use consistent stroke weights
- Maintain grid alignment
- Test at small sizes
- Provide accessible alternatives
- Use appropriate color coding
- Follow platform conventions
- Keep designs simple and clear

### ❌ DON'T
- Mix different icon styles
- Use overly complex details
- Ignore accessibility standards
- Create platform-specific confusion
- Use colors as sole differentiator
- Scale icons non-proportionally
- Forget to test legibility
