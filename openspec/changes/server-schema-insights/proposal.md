## Why

Beyond a numeric health score, customers benefit from AI-generated **insights** about a base — notable observations ("Customers and Orders form a circular reference", "the Imports table has 40 fields and no descriptions") that point at specific tables/fields. These should regenerate when the schema changes significantly, stay current (stale observations archived automatically), reference entities for quick navigation, and be configurable via prompts.

## What Changes

- Introduce **insights**: AI-generated observations per base, stored as rows, each able to **tag** specific tables/fields (for sidebar links / visual reference, mirroring document tags).
- Insights are generated by a **`workflows` task** after a schema capture, but **only when significant schema changes are detected** (material diffs since the last insights run) — debiting credits.
- A **space-level prompt** with an optional **per-base override**; effective prompt resolves **base override → space prompt → system default**. Each base shows a **last-generated** date and a **re-run** option when the prompt has changed.
- On each generation, the system **re-evaluates prior active insights** for accuracy against the current schema; any no longer accurate are set to **`status = 'archived'`** and dropped from the default view.

## Capabilities

### New Capabilities
- `schema-insights`: the insights data model (per-Space rows + entity tags), the prompt config (space + per-base override), the accuracy/archival lifecycle, last-generated/re-run, and the engine read/config API.
- `schema-insights-generation`: the `workflows` AI task that generates insights on significant schema changes and re-evaluates/archives prior insights, with credit debiting and incremental gating.
