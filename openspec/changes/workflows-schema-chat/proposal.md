## Why

Chat replies are generated by Claude on the workflows Node runner (the established AI location — health scoring runs there too), keeping the `ANTHROPIC_API_KEY` out of the backup engine. The engine ([`server-schema-chat`](../server-schema-chat/)) assembles the metadata-only context + enqueues this task; the task calls Claude and writes the reply back.

## What Changes

- A Trigger.dev task `chat-respond` that, given `{spaceId, threadId, assistantMessageId, context, history, userMessage}`, calls Claude (model `claude-opus-4-8`) with a metadata-only system prompt + context, then POSTs `/chat/message-complete` to resolve the pending assistant message.
- Pure orchestration `runChatRespond(input, deps)` (inject `generate` + `postComplete`): builds the message list (history + new user turn), generates, completes; on generation error writes a friendly `error` reply so the UI stops waiting.
- Non-streaming for MVP (returns the whole reply, then writes it back). Sovereign-AI: only schema metadata + the user's own messages are sent.

## Capabilities
